# OpenCodeHX Beads seed import notes

This seed uses stable `external_key` values like `opencodehx-021`. Real Beads IDs are normally generated by `bd`, so Codex should create issues in dependency order and keep the external key in the description or a label until a scripted import path is chosen.

Recommended setup:

```sh
bd init --quiet
bd setup codex
bd prime
```

Recommended labels are already included in `opencodehx-beads-backlog.seed.jsonl`.

Minimum first import set:

1. `opencodehx-000` epic
2. `opencodehx-001` source inventory
3. `opencodehx-002` genes-ts version verification
4. `opencodehx-003` workspace/Beads setup
5. `opencodehx-004` upstream OpenCode oracle manifest
6. `opencodehx-005` genes-ts NodeNext minimal app smoke
7. `opencodehx-045` genes-ts limitation ledger

After those are created, use `bd ready --json` and let Codex claim unblocked tasks.
